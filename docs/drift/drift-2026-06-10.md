### 5. HRV CV threshold doc says "CV > 10% flagged" — code was recalibrated away from the flat cutoff
> features.md line 57: **"Elevated HRV CV is an early marker of non-functional overreaching ... (Flatt & Esco 2016; CV > 10% flagged)."**

Code (`src/lib/training.ts:124–195`) was **recalibrated 2026-05-29**: the flat 10% cutoff is now only a fallback (`FLAT_HRV_CV_THRESHOLD = 10`). The live logic uses a *personalized* threshold — the user's own rolling-window CV mean + 1 SD (`rollingHrvCvBaseline()`, `hrvCvThreshold()`, `isHrvCvElevated()`). The in-code comment explains why: for a low-HRV individual (~20 ms mean) ordinary night-to-night variation mechanically yields 25–40% CV, so a flat 10% fires every day. **features.md's "CV > 10% flagged" is now factually wrong for the shipped behavior.** (This is the calibration that was making the training call read "Easy" every day.)

**Action:** Update features.md line 57 to describe the personalized CV baseline, not the flat 10%.
